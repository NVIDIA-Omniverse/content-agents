# Experimental Qwen3.6 35B On H100

This path follows the same Hyperstack H100 storage setup as the validated
Qwen3.5 35B path, with one extra writable workspace mount required by this
image.

```bash
ssh wu-vlm-h100-hs \
  'sudo mkdir -p /ephemeral/nim /ephemeral/nim-workspace && \
   sudo chown -R $(id -un):$(id -gn) /ephemeral/nim /ephemeral/nim-workspace'
```

Start the container:

```bash
ssh wu-vlm-h100-hs \
  'docker rm -f qwen36b-nim >/dev/null 2>&1 || true; \
   docker run -d --name qwen36b-nim --gpus all --shm-size=32GB \
     --env-file ~/.ngc-nim.env \
     -e NIM_SERVER_PORT=8000 \
     -e NIM_MAX_IMAGES_PER_PROMPT=20 \
     -e HOME=/tmp \
     -e USER=$(id -un) \
     -e VLLM_ENABLE_CUDA_COMPATIBILITY=1 \
     -e VLLM_WORKER_MULTIPROC_METHOD=spawn \
     -e PYTHONMULTIPROCESSING_START_METHOD=spawn \
     -u $(id -u) \
     -v /ephemeral/nim:/opt/nim/.cache \
     -v /ephemeral/nim-workspace:/opt/nim/workspace \
     -p 8000:8000 \
     nvcr.io/nim/qwen/qwen3.6-35b-a3b:1.7.0-variant'
```

Readiness can be slow because this image runs SGLang with DeepGEMM JIT warmup
and CUDA graph capture. Use the exact served model ID from `/v1/models`:

```bash
curl -fsS http://localhost:8003/v1/models
curl -fsS http://localhost:8003/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"Qwen/Qwen3.6-35B-A3B","messages":[{"role":"user","content":"Reply with exactly: brev-ok"}],"max_tokens":32,"temperature":0,"chat_template_kwargs":{"enable_thinking":false}}'
```

Before using this endpoint for an agent, run an image payload smoke that matches
the agent. Public image URLs can pass while base64 data URLs fail.
